# PokeApp

[![CI](https://github.com/beanthemoonman/pokeapp/actions/workflows/ci.yml/badge.svg)](https://github.com/beanthemoonman/pokeapp/actions/workflows/ci.yml)

## Purpose

In the same vein as a similar project I wrote last year https://github.com/beanthemoonman/photos,
a lens into understanding what's possible with agentic programming. This time using Claude Design as
well as Claude Code for implementation.

## Why Pokemon?

PokeApi: https://pokeapi.co/docs/v2 seemed very cool and turned out to be... very cool. Something to
take away from this besides all the slop based learning is that PokeApi is a wonderful tool to use
when you need to make a REST API based project in an area where everything else is unfamiliar.

## Scope

An Android Phone and TV app that provides basic helper functionality when playing through the games,
or settling arguments with people who misremember facts.

## Lessons learned

- Tools like Claude Design have made it very easy to vibe code something that looks professional
  instead of what you get is what you get
- Wowzers has price gotten out of hand. This was done with a Claude Pro subscription but I tried to
  keep notes of cost in .claude/usage-log.txt
- Big changes have less degradation in performance than before. I wouldn't attribute this as much
  to "oh my god Sonnet 4 is nothing compared to Opus 4.8" as I would that the tools around LLMs have
  gotten so much better. I'm hoping what this means in the long term is that with tools like
  OpenCode
  and cheaper to run "open source" models an affordable, profitable, mutually beneficial coexistence
  with these very helpful tools is possible in the near future.
- Apps are harder to share with others than websites. An unfortunate choice to make for something
  that was meant to be shared for the sake of discussion.
